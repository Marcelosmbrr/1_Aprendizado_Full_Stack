Geral:
1- Index se comunicará, primeiro, com o Script AJAX, enviando para ele os dados a serem processados
2 - Tendo recebidos os dados do Index, o Script AJAX os enviará para os scripts PHP correspondentes, que farão as verificações dos dados
3 - Após verificados pelos scripts de verificação, os dados serão enviados para a classe Pessoa_DAO que por si mesmo instanciará a classe SQL_DAO e a utilizará para processar os dados
4 - O retorno da execução das queries, pela classe SQL_DAO, será evidentemente para a classe Pessoa_DAO, que a instânciou, e que fará a seleção dos dados a serem retornados para o Script de verificação
5 - Os scripts de verificação recebem os dados, e os envia para o Script Ajax com um "echo" da função "json_encode()" que irá serializar os dados processados
6 - Os dados serializados serão recuperados pelo Script Ajax com o método .done() e impressos com HTML no View

Ordem de funcionamento:

Camada View:
1 - Para que o programa cumpra com sua finalidade, deve dispor dos dados que o usuário necessita
2 - O usuário pode inserir dados via input, e estes devem ser processados

Script Ajax:
1 - Terá funções que serão ativadas a partir de eventos realizados no View, pelo usuário
2 - Essas funções receberão os inputs do usuário e os enviará para os scripts php de verificação correspondentes

Scripts de verificação:
1 - Receberão os dados do Script AJAX
2 - Aplicarão os filtros de dados, impedindo possíveis ataques, como injeção de SQL
3 - Enviarão os dados filtrados para a classe Pessoa_DAO, por meio de instâncias SINGLETON dessa classe, é claro

Classe Pessoa_DAO:
1 - Irá receber a query e, por si mesmo, por meio de instâncias, irá enviar os dados para processamento pela classe SQL_DAO
2 - A classe Pessoa_DAO não cria novas instâncias necessariamente, porque aqui entra em funcionamento o padrão SINGLETON
3 - A classe Usuário terá como atributos privados os campos existentes dos registros da tabela Usuários, do banco
4 - A classe terá, naturalmente, os métodos getter e setters 

Classe SQL_DAO:
1 - A camada que irá fundir os dados com o banco será a classe SQL_DAO
2 - Irá utilizar a classe PDO a partir de instâncias SINGLETON da classe Conexao_instance
4 - A execução da query pela classe SQL_DAO será retornada para a classe Pessoa_DAO, que terá o objeto da classe SQL_DAO instanciado

Classe Pessoa_DAO:
1 - Novamente esta classe entra em cena, pois recebe os dados processados pela classe SQL_DAO - claro, se não for um select, acho que será retornado apenas um true ou false
2- Sendo os "dados", registros, ou apenas confirmações do processamento, os dados são retornados novamente para o Script de verificação

Script de verificação:
1 - Recebe os dados processados por todas as camadas anteriores, e retornados pela classe Pessoa_DAO
2- Envia os dados via JSON dentro de um array associativo, com o método json_encode(), novamente para o Script Ajax

Script Ajax:
1 - Os dados são recuperados neste script, com o método .done() e impressos no view com, provavelmente, o método .prepend()

DETALHES:
A condição de execução do SELECT universal para impressão de todos os registros, na camada View, será o carregamento do body, com "onload"
